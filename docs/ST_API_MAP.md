## 1. `generateRaw`

**Defined:** `public/script.js` — named export
**Signature:**
```javascript
export async function generateRaw({
    prompt = '',
    api = null,
    instructOverride = false,
    quietToLoud = false,
    systemPrompt = '',
    responseLength = null,
    trimNames = true,
    prefill = '',
```

**Returns:** `string` — the cleaned response text (via `cleanUpMessage()`). Throws if the API returns an error or produces no output.

**Usage in extension:**
```javascript
import { generateRaw } from '../../../script.js';

const result = await generateRaw({ prompt: myPrompt });
```

**Notes:**
- Always `await` — fully async, hits the live API/model.
- `api: null` defaults to `main_api` (whatever the user has selected).
- `systemPrompt` is separate from `prompt` — use it for instruction framing.
- `jsonSchema` enables structured JSON output where the API supports it.
- No streaming — returns the complete response string.

## 3. `saveCharacterDebounced`

**Defined:** `public/script.js:442`
```javascript
export const saveCharacterDebounced = debounce(
```

**Full trace:**
1. Triggers click on `#create_button` — the submit button of `#form_create`.
2. `#form_create` submit → `createOrEditCharacter(e.originalEvent)` (`script.js:11128`).
3. `createOrEditCharacter` reads all fields from the DOM form, builds `FormData`, POSTs to `/api/characters/edit`.

**Why NOT to use this from an extension:** The DOM form must be pre-populated with every character field. Extensions don't control the form state. **Use a direct `fetch` to `/api/characters/edit` instead** — see §4.

## 4. `POST /api/characters/edit`

**Handler:** `src/endpoints/characters.js` ~line 1091
**Content-Type:** `multipart/form-data`

**Key behavior — `charaFormatData(request.body)` (`characters.js:566`):**
- Starts from `tryParse(data.json_data) || {}` as the base object.
- Then applies `_.set(char, 'name', data.ch_name)`, `_.set(char, 'description', data.description || '')`, etc. for every V1/V2 field.
- **Missing fields become `''`** — this is fully destructive. If `personality` is not in the FormData body, `personality` will be wiped.

**Safe pattern — pass `json_data` as base:**
```javascript
const char = structuredClone(context.characters[this_chid]);
char.description = newDescription;

const formData = new FormData();
formData.append('json_data', JSON.stringify(char));   // base object
formData.append('avatar_url', char.avatar);           // required by middleware
formData.append('ch_name', char.name);                // required (validated server-side)
formData.append('description', char.description);
formData.append('personality', char.personality ?? '');
formData.append('scenario', char.scenario ?? '');
formData.append('first_mes', char.first_mes ?? '');
formData.append('mes_example', char.mes_example ?? '');
formData.append('creator_notes', char.data?.creator_notes ?? '');
formData.append('system_prompt', char.data?.system_prompt ?? '');
formData.append('post_history_instructions', char.data?.post_history_instructions ?? '');
formData.append('tags', JSON.stringify(char.tags ?? []));
formData.append('creator', char.data?.creator ?? '');
formData.append('character_version', char.data?.character_version ?? '');
formData.append('chat', char.chat);
formData.append('create_date', char.create_date);

// IMPORTANT: do NOT set Content-Type manually — browser sets multipart boundary
const headers = getRequestHeaders();
delete headers['Content-Type'];

const res = await fetch('/api/characters/edit', {
    method: 'POST',
    headers,
    body: formData,
});
if (!res.ok) throw new Error(`Character save failed: ${res.status}`);
```

**Response:** `200` on success, `400` for invalid name, `500` for server error.

## 5. `POST /api/chats/save`

**Handler:** `src/endpoints/chats.js` ~line 469
**Content-Type:** `application/json`

**Body shape:**
```javascript
{
    ch_name: string,        // character name (characters[this_chid].name)
    avatar_url: string,     // avatar filename e.g. "char.png"
    file_name: string,      // chat filename WITHOUT .jsonl suffix
    chat: ChatObject[],     // array — first element is metadata header
    force: boolean          // true = skip integrity check
}
```

**`chat` array:**
- `chat[0]` — metadata header object (see §7)
- `chat[1..N]` — regular message objects

**`force` flag** — controls `skipIntegrityCheck` in `trySaveChat()` (`chats.js:456`):
- `false` (default): validates `chat[0].chat_metadata.integrity` against on-disk slug. Throws `IntegrityMismatchError` (HTTP 400, `{ error: 'integrity' }`) if mismatch.
- `true`: skips integrity check entirely, always writes.
- For new chapter files (new filename, file doesn't exist yet) the check would trivially pass either way. Use `force: true` to be safe.

**Response:** `{ ok: true }` on success. `400` if `chat` is not an array.

```javascript
const res = await fetch('/api/chats/save', {
    method: 'POST',
    headers: getRequestHeaders(),
    body: JSON.stringify({
        ch_name: char.name,
        avatar_url: char.avatar,
        file_name: chapterName,
        chat: [metadataHeader, ...lastNMessages],
        force: true,
    }),
});
if (!res.ok) throw new Error(`Chat save failed: ${res.status}`);
```

## 6. `openCharacterChat`

**Defined:** `public/script.js:7522` — named export

**Full body:**
```javascript
export async function openCharacterChat(file_name) {
    await waitUntilCondition(() => !isChatSaving, debounce_timeout.extended, 10);
    await clearChat({ clearData: true });
    characters[this_chid].chat = file_name;
    chat_metadata = {};
    await getChat();
    $('#selected_chat_pole').val(file_name);
    await createOrEditCharacter(new CustomEvent('newChat'));
}
```

**Step by step:**
1. Waits for any in-progress chat save to finish (`isChatSaving` flag).
2. Clears the current chat UI and in-memory data.
3. Updates `characters[this_chid].chat = file_name` in memory.
4. Resets `chat_metadata` to `{}`.
5. Calls `getChat()` — fetches chat messages from `POST /api/chats/get` and populates `chat[]`.
6. Updates the `#selected_chat_pole` UI element.
7. Calls `createOrEditCharacter(new CustomEvent('newChat'))` — this POSTs to `/api/characters/edit` using current DOM form state, persisting the updated `.chat` pointer on the character card.

**Critical sequencing constraint:** Step 7 POSTs to `/api/characters/edit`. If Chapterize's own character save (§4) has not completed first, this call will race against it and may overwrite the updated `description` with stale values from the DOM form. **Always `await` the character save before calling `openCharacterChat`.**

**Import:**
```javascript
import { openCharacterChat } from '../../../script.js';
```

## 7. `context.chat` — array shape

**Accessed via:** `SillyTavern.getContext().chat`

**Important:** In memory, `context.chat` is an array of **regular message objects only**. The metadata header is stored separately in the module-level `chat_metadata` variable, not as `chat[0]`.

When ST **saves** a chat to disk (`script.js:7202`), it prepends the header:
```javascript
const chatHeader = { chat_metadata: metadata, user_name: 'unused', character_name: 'unused' };
// then saves [chatHeader, ...trimmedChat]
```

When ST **loads** a chat, it reads the JSONL, `shift()`s the first element into `chat_metadata`, and the remaining messages become `chat[]`.

**For building the new chat file**, construct the header explicitly from `chat_metadata`:
```javascript
const header = {
    chat_metadata: structuredClone(chat_metadata),  // or getContext().chat_metadata if exposed
    user_name: 'unused',
    character_name: 'unused',
};
header.chat_metadata.lastInContextMessageId = 0;
header.chat_metadata.integrity = crypto.randomUUID();
header.chat_metadata.tainted = false;
```

If `getContext()` does not expose `chat_metadata`, fall back to cloning a suitable base:
```javascript
const header = {
    chat_metadata: {
        lastInContextMessageId: 0,
        integrity: crypto.randomUUID(),
        tainted: false,
    },
    user_name: 'unused',
    character_name: 'unused',
};
```

**Regular message object fields:**
```javascript
{
    name: string,           // speaker name
    mes: string,            // message text
    is_user: boolean,       // true = user, false = AI
    is_system: boolean,     // true = system/narrator (filter these out of transcript)
    send_date: number,      // timestamp
    gen_started: Date,      // AI generation start (AI messages)
    gen_finished: Date,     // AI generation end (AI messages)
    swipe_id: number,       // active swipe index
    swipes: string[],       // swipe alternatives
    swipe_info: object[],   // swipe metadata
    extra: {
        api: string,
        model: string,
        token_count: number,
        type: string,       // system_message_types (e.g. 'narrator')
        // ...
    },
    // optional:
    title: string,
    force_avatar: string,
    original_avatar: string,
}
```

**Filtering for transcript:**
```javascript
const messages = context.chat.filter(m => !m.is_system);
// No need to skip chat[0] — in memory there is no header element
```

## 8. `context.characters[characterId]`

**Accessed via:**
```javascript
const context = SillyTavern.getContext();
const char = context.characters[context.characterId];
```

**All fields relevant to FormData build:**

| Field | Location on object | FormData key |
|---|---|---|
| Name | `char.name` | `ch_name` |
| Avatar filename | `char.avatar` | `avatar_url` |
| Description | `char.description` | `description` |
| Personality | `char.personality` | `personality` |
| Scenario | `char.scenario` | `scenario` |
| First message | `char.first_mes` | `first_mes` |
| Message examples | `char.mes_example` | `mes_example` |
| Creator notes | `char.data.creator_notes` | `creator_notes` |
| System prompt | `char.data.system_prompt` | `system_prompt` |
| Post-history instructions | `char.data.post_history_instructions` | `post_history_instructions` |
| Tags | `char.tags` (array) | `tags` (JSON stringified) |
| Creator | `char.data.creator` | `creator` |
| Character version | `char.data.character_version` | `character_version` |
| Current chat name | `char.chat` | `chat` |
| Creation date | `char.create_date` | `create_date` |
| Alternate greetings | `char.data.alternate_greetings` | `alternate_greetings` (JSON) |
| Extensions blob | `char.data.extensions` | passed via `json_data` |

**`json_data`:** Not a live field on the in-memory character object. It's the raw JSON string sent in FormData to give `charaFormatData` a base object. Build it as `JSON.stringify(char)` — the server merges it with the explicit fields, explicit fields winning.

**V2 spec container:** `char.data` holds V2 fields. `char.data.extensions` is an object for per-extension storage (deep-merged server-side).

## 9. Button injection pattern

**How ST loads extensions:**
`extensions.js` appends a `<script type="module">` tag pointing to `{name}/index.js`. Top-level `await` runs at load time. No mandatory `init()` export — just execute at module scope.

**Standard pattern used by built-in extensions:**
```javascript
// index.js — top level

import { extension_settings, renderExtensionTemplateAsync } from '../../extensions.js';
import { saveSettingsDebounced, getRequestHeaders } from '../../../script.js';

const EXT_NAME = 'chapterize';

async function init() {
    // 1. Load and inject settings HTML
    const settingsHtml = await renderExtensionTemplateAsync(EXT_NAME, 'settings');
    $('#extensions_settings').append(settingsHtml);

    // 2. Inject button into chat send area
    const btn = $('<div id="chapterize_btn" class="list-group-item" title="Chapterize">'
        + '<span>Chapterize</span></div>');
    $('#extensionsMenu').append(btn);           // wand/sparkle (✨) button, bottom-left of chat

    // 3. Bind handler
    btn.on('click', onChapterizeClick);

    // 4. Init settings
    extension_settings[EXT_NAME] = Object.assign({
        turnsN: 4,
        selfcheck: true,
    }, extension_settings[EXT_NAME]);
}

await init();
```

**DOM attachment targets:**

| Target | Element ID/class | Used by |
|---|---|---|
| Extensions gear menu | `#extensionsMenu` | Most action extensions |
| Send bar (right of send) | `#send_but_sheld` | TTS, image gen buttons |
| Per-message buttons | `.extraMesButtons` (delegated) | Narrate, translate buttons |
| Extensions settings panel | `#extensions_settings` | Settings UI |

**jQuery is available globally** — no import needed. Both `$()` and `jQuery()` work.

**`renderExtensionTemplateAsync(extName, templateName)`** — loads `{extName}/{templateName}.html` via the ST template system. Use for settings panels. Not required for simple injected buttons.

## `getRequestHeaders()`

**Defined:** `public/script.js` — named export
**Returns:** `{ 'Content-Type': 'application/json', 'X-CSRF-Token': token }` (approximately)
**Required** on all `fetch` calls to ST server endpoints.
**Exception:** Do NOT include it (or delete `Content-Type` from it) when using `FormData` — the browser must set the multipart boundary itself.

```javascript
import { getRequestHeaders } from '../../../script.js';

// JSON fetch
fetch('/api/chats/save', {
    method: 'POST',
    headers: getRequestHeaders(),
    body: JSON.stringify(payload),
});

// FormData fetch
const headers = getRequestHeaders();
delete headers['Content-Type'];
fetch('/api/characters/edit', {
    method: 'POST',
    headers,
    body: formData,
});
```
